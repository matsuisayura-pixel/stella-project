# /learn — 教訓・知見の自動キャプチャ

非自明な問題を解決した直後に実行してください。
解決策を `docs/mistakes-log.md` と `AGENTS.md` に記録します。

## 実行フロー

1. 直近の会話から「非自明な解決策」を抽出
2. 以下の形式でフォーマット
3. `docs/mistakes-log.md` に追記
4. `AGENTS.md` の該当セクションに追記を提案
5. ユーザーに確認して保存

## 抽出する情報

- **問題**: 何が起きたか
- **原因**: なぜ起きたか
- **解決策**: どうやって解決したか（コード例あれば含む）
- **予防法**: 次回どうすれば避けられるか
- **カテゴリ**: bug / pattern / gotcha / architecture / workflow

## 出力先

```
docs/mistakes-log.md   → 詳細ログ（検索可能）
AGENTS.md              → 要約（毎セッション参照される）
```

## 実行手順

以下のプロセスを実行してください:

1. 直近の会話を振り返り、非自明だった解決策を3つまで特定する
2. 各解決策を以下のテンプレートで `docs/mistakes-log.md` に追記する:

```markdown
## [YYYY-MM-DD] [カテゴリ] タイトル

**問題**: ...
**原因**: ...
**解決策**: ...
**予防法**: ...
```

3. AGENTS.md の「絶対に繰り返してはいけない間違い」または「有効だったパターン」テーブルに1行追記を提案する
4. ユーザーに「保存しますか？」と確認する
5. 承認されたら Write ツールで保存する

## 注意

- 明らかな（自明な）解決策は記録しない
- コードスニペットは必要最小限
- 1エントリは200文字以内
- 記録するのは「次のClaudeが見たら役立つ」情報のみ

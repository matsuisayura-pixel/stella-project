# story-pattern-delayed-rescue
## 物語YOUTUBE台本パターン③「遅れて届く救い型」

---

## このパターンが解決する問題

A型テンプレートの弱点：救済がすべて「事前に準備済み」で主人公の前に置かれる。
「家賃全額支払済み・奨学金手配済み・解雇も完了」——これは主人公に**何の選択肢も与えない**。

**100点に必要な核心：**
> 主人公は一度、失う。
> そこから自分の足で動こうとする意志を見せた後に、初めて救いが届く。
> 救いは「全解決」ではなく「扉を開ける」だけ。残りは主人公自身が歩く。

---

## パターン構造（全10幕）

### 幕1：冒頭フック
- A型同様、衝撃的なシーンから始める
- ただし「最後に逆転する」という予感を与えすぎない
- 冒頭は「逆転の予感」より「現在の絶望の深さ」を優先する

### 幕2：主人公の日常（失う前の「持っていたもの」を描く）
- **このパターン固有の必須要素**：
  - 主人公がまだ「何か」を持っている状態から始める
  - 家・仕事・関係性・誇り——何でもいいが、1つは持っている
  - それが後に失われることで、「落差」が生まれる

### 幕3：屈辱①（外部からの攻撃）
- 組織・制度・人物からの理不尽な打撃
- 主人公はまだ抵抗できる体力がある

### 幕4：善行（まだ余裕がある状態で）
- **このパターンの善行の特徴**：
  - 「最後のお金」ではなく「余裕はないが、まだある状態」で助ける
  - 「後で返ってくる」という計算は全くない
  - むしろ「損をした」という感覚が残る
- 例：「今月の食費が削られる。でも、まあいいか」程度

### 幕5：屈辱②（善行のコスト）
- 善行によって余裕がさらに減る

### 幕6：屈辱③（決定打）
- 主人公が「持っていたもの」を失う
- **このパターン固有の必須要素**：
  - 失うシーンを丁寧に描く（短くしない）
  - 失った後の「静寂」を描く（泣かない・動けない）
  - 視聴者が「もう終わりか」と感じるほど深く落とす

### 幕7：主人公の「自力の試み」（このパターン最重要幕）
- **A型との最大の違いがここ**
- 救済が来る前に、主人公が自分で動く
- **必須描写**：
  ```
  [何を試みるか]
  - 知人への相談（断られる or 小さな助けをもらう）
  - 行政窓口への相談（複雑な手続きと長い待ち時間）
  - 新しいバイト・仕事の面接（不採用 or 採用されるが間に合わない）
  - 自分の持ち物を売る（スマホ・家具・思い出の品）
  ```
- このどれかを実際に行動に移す
- **全部うまくいかなくていい**：重要なのは「諦めない意志」
- 助けを求めること自体が「弱さ」ではなく「強さ」として描く

### 幕8：ツイスト（救いが「届く」）
- **「来る」ではなく「届く」**：主人公が動いたから手が届く距離に来た
- ツイストは幕7の「自力の試み」と連動している
  - 例：行政窓口に相談に行ったとき、同じ窓口にツイストの人物がいた
  - 例：持ち物を売りに行った店で、ツイストの人物と再会した
- **救済の範囲を「全解決」にしない**：
  ```
  NG: 「家賃36万全額・奨学金月15万・解雇も完了」（一括全解決）
  OK: 「まず今月の家賃だけ」「一つの扉を開ける」
      「あとは自分で歩ける分だけ、道筋を示す」
  ```

### 幕9：制裁（静かに・間接的に）
- 主人公が直接立ち会わない制裁も効果的
- 後日談として「あの人はこうなったらしい」程度でも十分

### 幕10：恩送り（主人公が「歩く」姿で終わる）
- **このパターンの恩送りの核心**：
  - 主人公はまだ完全には回復していない
  - それでも、次の誰かに手を差し伸べる
  - 「余裕があるから助ける」のではなく「余裕がなくても助ける」
- **エンディングの型**：
  ```
  主人公はまだ [具体的な問題] を抱えている。
  [具体的な数字] の借金が残っている。
  それでも、今日、[具体的な場所] で [誰か] を見かけたとき——
  足が止まっていた。
  ```
- 「解決した人が助ける」ではなく「まだ途中の人が助ける」ことが感動の核

## キャラクター設計

### 主人公の「自力の試み」に出てくる人物たち
これらは脇役だが、重要な機能を持つ：

```
行政窓口の担当者:
  → 冷たくではなく「疲れた顔で」対応する
  → 「制度上できることとできないことがある」という現実を示す

断った知人:
  → 悪人ではない。自分も余裕がないだけ
  → 「ごめん、本当にごめん」と言いながら断る

面接官:
  → 履歴書の空白期間を指摘する
  → 丁寧に「今回は採用が難しい」と告げる
```

### 恩人（ツイストの人物）
- 「全部解決する人」ではなく「一つの扉を開ける人」
- 自分も過去に似た経験をしていた（ただし完全一致の鏡像にしない）
- 「あとは自分で」という言葉を必ず言わせる
- 完璧な救済者ではなく、「同じ道を先に歩いた人」として描く

---

## 時系列の設計（重要）

このパターンは時系列が鍵：

```
Day 1-3:   日常 → 打撃①
Day 4-7:   善行 → 打撃②
Day 8-10:  決定打 → 喪失
Day 11-15: [主人公の自力の試み期間] ← このパターン最重要
Day 16:    ツイスト（届く）
Day 17+:   制裁・恩送り

※ 「自力の試み」に最低4日は使う
※ A型の5日間（善行→立退）より長い時間軸を使う
```

## リアリティ・チェックリスト

- [ ] 主人公が救済前に「自力で何か」を試みているか（最低1つの行動）
- [ ] 救済が「全解決」ではなく「扉を開ける」にとどまっているか
- [ ] 恩送りシーンで主人公はまだ「完全復活」していないか
- [ ] ツイストが主人公の行動と連動しているか（受動的ではないか）
- [ ] 自力の試みで出会う人物が「悪人」ではなく「余裕のない普通の人」か
- [ ] エンディングで「それだけのことだった」を使っていないか
- [ ] 恩送りシーンが400字以上あるか

```

```

例：
```
/story-pattern-delayed-rescue
テーマ: 工場をリストラされた40代男性が、ホームレスの老人に最後の弁当を渡した
主人公: 42歳男性・元工場勤務・妻と娘あり・再就職活動中
舞台: ハローワーク・深夜の公園・安アパート・コンビニ
```

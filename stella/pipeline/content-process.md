# ステラ コンテンツ生成プロセス 完全設計書

---

## 全体フロー図

```
【PHASE A：インプット】
  動画 / 音声 / PDF / ウェブページ
        ↓
【PHASE B：素材化】
  文字起こし・テキスト抽出 → knowledge-base に保存
        ↓
【PHASE C：精査・整合性チェック】
  ① エッセンス抽出（概念・教えの核心）
  ② 矛盾検出（複数先生の教えを照合）
  ③ 多次元統合（矛盾を「季節」で解決）
        ↓
【PHASE D：リサーチ補強】
  /research-free または /mega-research-plus
  → 最新情報・エビデンスを追加
        ↓
【PHASE E：ステラ化（著作権処理）】
  /taiyo-style → ステラ口調に完全変換
  /taiyo-analyzer → 品質チェック（70点以上）
        ↓
【PHASE F：コンテンツ出力】
  note記事 / 音声スクリプト / （後でショート動画）
```

---

## PHASE A：インプット（素材の受け取り）

### 対応できる素材の種類

| 素材 | 処理方法 |
|------|---------|
| 動画（MP4等） | 音声を聞いて要点をメモ → 文字起こし |
| 音声（MP3等） | 同上 |
| PDF | テキストをコピー or 貼り付け |
| ウェブページ | URLを共有 → 内容を取得 |
| テキスト | そのまま貼り付け |

### 素材を渡す際の指示テンプレート

```
以下の素材を knowledge-base に追加してください。

【素材の種類】動画の文字起こし / PDF / テキスト / URL
【先生の属性】行動派 / 受容派 / スピリチュアル系 / 自己啓発系
【主なテーマ】例：感情の解放、引き寄せ、人間関係
【素材の内容】
↓ここに貼る↓

```

---

## PHASE B：素材化（knowledge-base への保存）

### 保存フォーマット（自動的に以下の形式で整理）

```markdown
# [タイトル]

## メタ情報
- 素材種別：動画 / 音声 / PDF / Web
- 先生の属性：行動派 / 受容派（名前は記録しない）
- テーマタグ：#感情 #引き寄せ #人間関係 など
- 追加日：YYYY-MM-DD

## 原文エッセンス（要約・引用）
→ 素材の核心を箇条書きで抽出

## 概念の核心（What）
→ この教えが言っている「本質的な主張」は何か

## 対象読者のフェーズ
→ 行動の季節向け / 受容の季節向け / どちらにも使える

## 矛盾可能性メモ
→ 他の先生の教えと矛盾しそうな箇所を記録

## ステラ変換メモ（後で使う）
→ ステラの口調で言い換えたフレーズ案
```

## PHASE C：精査・整合性チェック

### ステップ 1：エッセンス抽出

素材を読んで「**概念の核心（What）**」だけを取り出す。

```
この素材から：
1. 最も重要な主張を1文で
2. 読者が得られる気づきを3つ
3. 使えるフレーズ・比喩を3つ
を抽出してください。
```

### ステップ 2：矛盾検出

複数の先生の教えを照合する。

```
矛盾パターン例：

A先生（行動派）：「感情は無視して行動を起こせ」
B先生（受容派）：「感情を感じきることが変化の鍵だ」

→ これは「矛盾」ではなく「季節の違い」として統合する
```

### ステップ 3：多次元統合ロジックで解決

矛盾を「どちらが正しいか」ではなく「今の読者はどの季節か」で解決。

```
【統合テンプレート】

A先生とB先生の教えが矛盾する場合：

「行動の季節（春・夏）にいる人へ」
→ A先生のアプローチ（行動重視）を採用

「受容の季節（秋・冬）にいる人へ」
→ B先生のアプローチ（感じる・休む）を採用

どちらの季節にいるかは、読者のフェーズで判断する。
ステラはこの「多次元統合ロジック」を使って、
「どちらも正しい、ただし今のあなたには〇〇が合っている」と伝える。
```

## PHASE D：リサーチ補強

素材の内容に最新情報・エビデンスを追加する。

### APIなし（今すぐ使える）

```
/research-free

「[テーマ]」について：
・最新の心理学・脳科学の知見
・2026年時点のトレンド
・日本語コミュニティ（Zenn・Qiita・はてブ）での反応
を調査してください。
```

### APIあり（後で使える）

```
/mega-research-plus

「[テーマ]」について8ソース統合リサーチを実行してください。
```

## PHASE E：ステラ化（著作権処理）

これが最重要フェーズ。「誰の教えか分からない、ステラの独自コンテンツ」に変換する。

### ステップ 1：概念の抽象化

```
【やること】
原文の「具体的なエピソード・固有表現・独自ワード」を取り除く。
「概念・原理・普遍的な気づき」だけを残す。

【例】
NG（原文のまま）：「○○メソッドの第3ステップ、感情のデトックス法」
OK（概念化）：「感情を手放すための実践」
```

### ステップ 2：ステラ口調への変換

```
/taiyo-style

以下の内容をステラ（Stella）の口調で書き直してください。

ルール：
・一人称「私」、読者「あなた」
・語尾「〜ですね」「〜なのです」
・専門用語は比喩に変換（character.mdの変換辞書を使用）
・講師名・固有メソッド名は一切出さない
・「かつての賢者はこう説いていました」等の形で引用
・疑問形33%：読者に問いかけながら進める
・恐怖→希望の流れで構成する

【変換する内容】
↓ここに貼る↓
```

### ステップ 3：品質チェック

```
/taiyo-analyzer

以下のコンテンツを6次元でスコアリングしてください。

チェックポイント：
□ 語尾・リズムが一貫しているか
□ 固有名詞・メソッド名が残っていないか
□ ステラらしい比喩が使われているか
□ CTAが挿入されているか
□ 70点以上か（未満なら再変換）

【コンテンツ】
↓ここに貼る↓
```

## PHASE F：コンテンツ出力

品質チェックを通過したら、各媒体に展開。

### note記事（優先①）
→ `content/note-template.md` の構成に当てはめる

### 音声スクリプト（優先②）
→ `content/voice-template.md` の構成に当てはめる
→ `/story-pattern-inner-conflict` でストーリー化

### ショート動画（後回し）
→ 後で `/shorts-create` を使って自動生成

---

## 著作権・独自性チェックリスト（最終確認）

コンテンツを公開する前に必ず確認：

```
【著作権チェック】
□ 特定の先生名・ブランド名が一切含まれていないか
□ 固有のメソッド名・商標が含まれていないか
□ 原文のフレーズがそのまま残っていないか（最低70%以上書き換え）
□ ステラ独自の比喩・言い回しに変換されているか

【独自性チェック】
□ 「ステラらしさ」が出ているか（穏やか・断定・多次元統合）
□ 複数の教えが統合されているか（1人の先生の教えだけに偏っていないか）
□ ステラの口調（語尾・一人称）が一貫しているか
□ CTAが正しく挿入されているか
```

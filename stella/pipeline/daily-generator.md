# 毎日のコンテンツ生成手順（スキル活用版）

## 使用スキルの流れ

```
テーマ選定
    ↓
/keyword-mega-extractor  ← SEOキーワード抽出
    ↓
/note-research           ← note売れ筋・競合分析
    ↓
/taiyo-style-headline    ← タイトル候補12パターン生成
    ↓
/note-marketing          ← 記事構成・ファネル設計
    ↓（素材を注入）
/taiyo-style             ← 高成約文章に変換（疑問形33%・恐怖→希望）
    ↓
/taiyo-analyzer          ← 6次元スコアリング → 70点未満は再生成
    ↓
/story-pattern-inner-conflict  ← ストーリー要素を追加（音声向け）
    ↓
CTA自動挿入（cta-templates.mdから選択）
    ↓
完成（note記事 + 音声スクリプト）
```

---

## Step 1：テーマ＆キーワード選定

```
/keyword-mega-extractor

プロンプト：
「[テーマ]」についてステラ（スピリチュアル系AIキャラ）の
note記事・音声コンテンツ向けに以下を抽出してください。
- core_keywords（5個）
- niche（競合少ないニッチキーワード）
- rising_2026（急上昇中のキーワード）
```

---

## Step 2：note売れ筋調査

```
/note-research

プロンプト：
「[テーマ]」でnoteの売れ筋記事を調査し、
・人気記事のタイトルパターン
・コンテンツ構成の傾向
・読者が反応するフック
を教えてください。
```

---

## Step 3：タイトル生成

```
/taiyo-style-headline

プロンプト：
テーマ：[テーマ]
ターゲット：[悩みを持つ読者像]
12パターンのタイトル候補を生成してください。
```

---

## Step 4：note記事生成

```
/note-marketing

プロンプト：
ステラ（Stella）として以下の条件で記事を書いてください。

【テーマ】[テーマ]
【タイトル】[Step 3で選んだタイトル]
【素材】[knowledge-baseから抜き出したエッセンス]
【読者の悩み】[具体的な悩み]
【構成】
1. つかみ（150字）：読者の悩みへの深い共感
2. 問題の本質（300字）：多次元統合ロジックで解説
3. 教えのエッセンス（600字）：素材をステラ口調に変換
4. 季節の診断（300字）：行動 or 受容の季節を示す
5. 具体的なアクション（300字）
6. まとめ（150字）：希望で締める
7. CTA（CTA-Aを挿入）

【必須ルール】
- 一人称：「私」、読者：「あなた」
- 語尾：「〜ですね」「〜なのです」
- 専門用語は比喩に変換（character.mdの変換辞書を使う）
- 講師名は「かつての賢者」等に置き換える
- 文字数：1500〜2000字
```

---

## Step 5：文章品質チェック（必須）

```
/taiyo-style

プロンプト：
以下のnote記事をtaiyoスタイルに変換してください。
[Step 4の記事を貼る]
```

```
/taiyo-analyzer

プロンプト：
以下の記事を6次元でスコアリングしてください。
70点未満の項目は改善案を提示してください。
[記事を貼る]
```

---

## Step 6：音声スクリプト生成

```
/story-pattern-inner-conflict

プロンプト：
Step 4の記事を元に、音声スクリプト（Voicy・3〜5分）向けに
「内的葛藤→気づき→解放」のストーリー構成で書き直してください。
ステラの口調（穏やか・語りかける）で。
末尾はCTA-Cを挿入。
```

---

## 完成チェックリスト

- [ ] note記事：1500〜2000字
- [ ] taiyo-analyzerスコア：70点以上
- [ ] CTA挿入済み（note: CTA-A）
- [ ] 音声スクリプト：3〜5分相当
- [ ] CTA挿入済み（音声: CTA-C）
- [ ] 講師名・固有名詞が消えている
- [ ] 専門用語が比喩に変換されている
